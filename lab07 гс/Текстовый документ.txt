Блок 1: Конструкция MODEL 
Для чего используется конструкция MODEL?
Она позволяет обрабатывать строки таблицы как элементы многомерного массива. Это нужно для выполнения сложных межбазовых вычислений (например, финансовых прогнозов или бюджетирования), которые сложно реализовать обычным SQL.

Поясните секцию ORDER BY конструкции MODEL.
Она задает порядок сортировки данных внутри каждой логической группы перед тем, как к ним будут применены правила. Это критично, когда результат вычисления одного правила зависит от порядка следования строк.

Поясните секцию PARTITION BY конструкции MODEL.
Разбивает весь набор данных на независимые логические блоки (разделы). Вычисления в одном разделе никак не влияют на данные в другом. В твоем случае мы делили данные по UserID.

Поясните секцию DIMENSIONS конструкции MODEL.
Определяет «координаты» или индексы массива. Это колонки, по которым мы идентифицируем конкретную ячейку в разделе (например, месяц Month или год).

Что такое мера (MEASURES) конструкции MODEL?
Это те колонки данных, над которыми проводятся вычисления. Если DIMENSIONS — это адрес ячейки, то MEASURES — это её содержимое (например, сумма аренды).

Что такое RULES?
Это список формул, описывающих, как вычислять значения мер. Правила могут обращаться к существующим ячейкам или создавать новые (как мы делали для 2026 года).

Что такое символьная, позиционная и смешанная нотации?
Это способы обращения к ячейкам в правилах:Символьная: обращение по значению индекса, например cost[Month = '01/01/2026'].Позиционная: использование функции CV() (Current Value), например cost[CV(Month)].Смешанная: комбинация обоих способов в одном правиле.

Что такое NESTED REFERENCES?
Это вложенные ссылки, когда одно правило в своей правой части вызывает результат другого правила или подзапроса для получения индекса.

Что такое REFERENCE MODEL?
Это вспомогательная (внешняя) таблица-справочник, которую можно подключить к основной MODEL только для чтения. Например, таблица курсов валют.

Для чего используется UNIQUE SINGLE REFERENCE?
Гарантирует, что по заданным координатам DIMENSIONS будет найдена ровно одна ячейка. Это предотвращает ошибки неоднозначности при вычислениях.Блок 2: Конструкция MATCH_RECOGNIZE

Для чего используется конструкция MATCH_RECOGNIZE?
Для распознавания образов (паттернов) в последовательностях строк. Она позволяет находить тренды, двойные продажи или аномалии в данных.

Что такое переменные шаблона в конструкции MATCH_RECOGNIZE?
Это логические имена (метки) для строк, которые удовлетворяют определенному условию. Например, мы помечали строку как UP, если текущая цена больше предыдущей.

Что такое шаблон (PATTERN) в конструкции MATCH_RECOGNIZE?
Это регулярное выражение, которое описывает последовательность переменных шаблона. Оно задает порядок, в котором должны идти события (например, STRT потом один или несколько UP).

Для чего используются меры (MEASURES) в конструкции MATCH_RECOGNIZE?
Определяют, какие значения будут выведены в итоговую таблицу для найденного совпадения. Здесь можно использовать функции FIRST, LAST, COUNT для всей найденной цепочки.

Для чего используется ключевая фраза AFTER MATCH в конструкции MATCH_RECOGNIZE?
Указывает, с какой строки продолжать поиск следующего совпадения после того, как текущее было найдено. Например, SKIP TO LAST UP2 говорит Oracle начать следующий поиск с конца только что найденного паттерна.
